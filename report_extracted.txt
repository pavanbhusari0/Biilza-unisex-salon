Abstract
The Complaint Management System is a dynamic and user-friendly web application designed to
provide users with a convenient platform to register, track, and manage complaints efficiently.
The system aims to simplify the complaint handling process by offering an interactive interface,
secure user authentication, and effective complaint tracking and management features.
The frontend of the website is developed using HTML, CSS, and JavaScript along with
Bootstrap, which ensures an attractive layout, responsive design, and smooth user interactions.
HTML is used to structure the web pages, CSS is applied for styling and visual appeal, and
JavaScript enables dynamic features such as form validation, real-time updates, and interactive
user experience.
The backend is developed using Python and the Django framework, which handles server-side
logic, user authentication, complaint submission workflows, status updates, and communication
between the frontend and database. SQLite is used as the database to store user details, complaint
records, status information, and other related data securely.
The website includes key features such as user registration and login, complaint submission,
complaint status tracking (Pending, In Progress, Resolved), and an admin module for managing
and resolving complaints. This project demonstrates the integration of frontend and backend
technologies to build a secure, scalable, and efficient complaint handling system.

INTRODUCTION
In today's modern world, technology plays a very important role in every field, including
education, organizations, and service sectors. Traditional complaint handling methods that rely
on manual processes such as written applications or verbal communication are now being
replaced by digital platforms to improve efficiency and accessibility.
Managing complaints manually — from recording issues to tracking their status and resolving
them — requires a significant amount of time and effort. It may also lead to errors, delays, and
lack of proper communication between users and administrators. A Complaint Management
System helps to simplify this process by making it faster, more organized, and efficient.
The main idea behind the Complaint Management System is to computerize complaint handling
operations such as user registration, complaint submission, status tracking, and resolution
management. It replaces traditional manual methods with an online platform where all complaint
records are stored securely in a database and can be accessed anytime.
This system allows users to register online, submit their complaints, and track the status of their
issues such as Pending, In Progress, or Resolved. The admin can view all complaints, update
their status, and manage the system effectively. This improves transparency, reduces workload,
and ensures that complaints are handled in a timely and efficient manner.
2.1 Motivation
In many organizations and institutions, complaint handling is still done manually through written
applications, phone calls, or verbal communication. This leads to several problems such as loss
of data, delays in processing, lack of proper tracking, and inefficient communication between
users and administrators.
 Users often face difficulty in knowing the status of their complaints, as there is no proper system
to track progress. They have to repeatedly visit offices or contact staff for updates, which is time-
consuming and inconvenient. Manual systems also increase the chances of complaints being
ignored or not resolved on time.
With the increasing use of digital technology, people now expect fast, transparent, and easily
accessible services. An online Complaint Management System provides a centralized platform
where users can submit complaints from anywhere and track their status in real time.
The need for a structured and automated system that can manage complaints efficiently, reduce
manual workload, and improve transparency was the main motivation behind developing this
system.
•
2.2 Problem Statement
• In many organizations and institutions, a large number of complaints are generated daily.
Managing these complaints manually through paper records or basic systems is time-
consuming, inefficient, and prone to errors. Important complaint data can be lost,
misplaced, or not recorded properly, leading to poor service and dissatisfaction among
users.
Many existing systems do not provide proper tracking and status updates for complaints. As a
result, users are unable to know whether their complaint is being processed or resolved. This lack
of transparency creates confusion and reduces trust in the system.

3.4 System Architecture (Complaint Management System)
The Complaint Management System follows a three-tier web architecture, which separates the
system into a presentation layer, a business logic layer, and a data layer.
1. Presentation Layer (Frontend)
This is the frontend part of the system that users interact with directly. It includes web pages
designed using HTML5, CSS3, Bootstrap, and JavaScript. Users use this layer to register, log in,
submit complaints, and track their complaint status. The interface is designed to be simple,
responsive, and user-friendly.
2. Application Layer (Backend — Django/Python)
Also known as the business logic layer. It handles all processing such as user authentication,
complaint submission, status updates, and communication between the frontend and database.
Python with the Django framework manages request handling, URL routing, and application
logic efficiently.
3. Database Layer (SQLite3)
This is the backend storage where all data is stored securely. Information such as user details,
complaint records, and complaint status is stored in an SQLite database. The application layer
interacts with the database using Django ORM to perform operations like storing, retrieving, and
updating data.

6.1.2 White Box Testing
--- PAGE 24 ---
White Box Testing (also known as Glass Box Testing) is a method of testing the internal
structure and logic of the code. In this approach, the tester examines the program’s internal
working and writes test cases to verify each part of the logic.
Objectives of White Box Testing
To verify that the code logic in all Django view functions related to complaint submission,
tracking, and management works correctly in all scenarios
To ensure that all conditions and loops used in complaint handling, status updates, and staff
assignment function properly
To identify any hidden logical or syntax errors in the Django backend code
Techniques Used
Statement Coverage: Every line of code related to complaint submission, status updates, user
authentication (login, register), complaint assignment, and feedback handling was executed at
least once during testing.
Branch Coverage: All conditional statements were tested, including different complaint statuses
(Pending, In Progress, Resolved), successful and failed login attempts, valid and invalid form
inputs, and admin vs user access flows.
Path Coverage: All possible execution paths through the system were tested thoroughly,
including the complete complaint lifecycle from submission to resolution, the complaint tracking
process, and feedback submission by users.
After performing white box testing, all internal modules of the system—including database ORM
operations, Django view logic, user authentication, complaint submission, status updates, staff
assignment, and feedback handling—worked correctly. No major syntax or logical errors were
found. The Online Complaint Management System successfully passed all white box tests,
confirming that the internal structure of the code is accurate, efficient, and reliable.

7. CONCLUSION
The Online Complaint Management System is a complete web-based solution designed to
simplify the process of registering, tracking, and resolving complaints efficiently. The system
successfully automates major functions such as complaint submission, status tracking, staff
assignment, and feedback collection, providing users with a fast, reliable, and transparent
platform.
Through this project, the issues present in traditional complaint handling methods—such as
manual record keeping, delayed responses, lack of transparency, and inefficient
communication—have been effectively addressed. The system offers a centralized platform
where users can easily register complaints and monitor their progress until resolution.
The Python and Django backend provides a secure and scalable foundation for the application,
while the SQLite database ensures reliable data storage. Django’s ORM enables efficient
database operations and maintainability. The frontend design focuses on simplicity,
responsiveness, and user-friendliness, ensuring a smooth experience across different devices.
The system supports multiple functionalities including complaint categorization, staff
assignment, status updates (Pending, In Progress, Resolved), and user feedback, making it a
comprehensive solution for complaint management. Its modular design also allows easy future
enhancements and scalability.
In conclusion, the Online Complaint Management System improves efficiency, transparency, and
user satisfaction in handling complaints. It demonstrates how modern web technologies can
streamline and optimize traditional processes, making them faster, more organized, and more
effective.
8 Future scope
Although the current version of the Online Complaint Management System fulfills all basic
requirements effectively, there is still scope for further improvement and enhancement. The
system can be extended with advanced features to make it more efficient and user-friendly.
The system can be upgraded by integrating real-time notification services such as SMS and email
alerts to inform users about complaint status updates. A mobile application for Android and iOS
can be developed to allow users to register and track complaints more conveniently. Artificial
Intelligence can be incorporated to automatically categorize complaints and prioritize them based
on urgency.
Additionally, a live chat or chatbot feature can be added to provide instant support to users. The
system can also include role-based dashboards with advanced analytics and reporting for
administrators to monitor performance and resolution rates. Integration with cloud platforms like
AWS or Google Cloud can improve scalability and accessibility.
In future, features like multilingual support, document verification, and a rating system for
service quality can further enhance user satisfaction. Overall, the system has strong potential to
evolve into a more intelligent, scalable, and user-centric complaint management solution.
